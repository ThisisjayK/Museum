Product philosophy

In my own words.

  1. Data first, but data needs context.
  2. Give heavy weight to user friction and feedback.
  3. Prioritize against the roadmap and business goals.
  4. Products should be personalized to the core.
  5. Users value honest products that do not overpromise.
  6. Understand user behavior deeply: time on product, interaction patterns.

Case study · Consumer

Koko — and knowing when to stop building

A mobile restaurant-saving app for three failure modes: capture friction (recommendations arrive in five-second windows), the enrichment gap (a name alone is useless later), and the retrieval gap (every competitor makes you remember to check). I built a mobile-first prototype with voice input, LLM parsing via Groq / Llama 3, place enrichment via OpenStreetMap / Nominatim, simulated geofence alerts, and notification suppression. Then I found the gating constraint: sharing a social reel into the app needs a native share-sheet extension because of CORS, so I made a deliberate stop decision rather than over-build.

  • Groq (Llama 3)
  • Foursquare Places
  • OpenStreetMap / Nominatim
  • Netlify

Case study · AI · In development

Synthia — turning interviews into signal, not soup

An AI user-interview synthesis tool. Three-stage pipeline: PII anonymization via AI (not regex), per-interview analysis against a fixed seven-bucket framework, and cross-interview synthesis. Key decisions: two-stage architecture for traceability, optional anonymization as a nudge rather than a mandate, paste-text only for V1 scope discipline, a manual synthesis trigger to avoid unnecessary API cost, and surfacing divergent signals rather than averaging them away.

  • React
  • Tailwind (Vite)
  • Supabase
  • Vercel
  • Anthropic API (Claude Sonnet)

Case study · Systems design

Human aging as a complex adaptive system

Applied five analytical frameworks (IPO, Iceberg Model, Causal Loop Diagram, System Archetypes, Stakeholder Network Analysis) to diagnose why aging interventions fail structurally.
